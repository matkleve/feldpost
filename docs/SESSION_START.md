# Session Start Prompt

Copy this into Cursor at the start of every session.

Before doing anything:

1. Read [AGENTS.md](../AGENTS.md) at the repository root completely.
2. Read [docs/specs/README.md](specs/README.md) — especially **Quick Orientation (Session Context)** and navigation to per-area READMEs under `docs/specs/`. Do not maintain or assume a global flat specs table; use each area’s local README as the index.
3. Run `git log --oneline -5` and read the output.
4. List any open GitHub issues labeled `task`. If GitHub is not reachable from your environment, ask the user to paste open `task` issues or note that this step is blocked.
5. Respond in 3–5 bullet points: what was recently worked on, the logical next step, and anything blocked or unclear.

Do not write any code yet. Wait for confirmation before proceeding.

## Maintainer: GitHub labels

Create exactly these labels on the repo (UI: **Settings → Labels**, or install the GitHub CLI and run `gh label create`): `idea` (`#C5DEF5`), `task` (`#0075CA`).
