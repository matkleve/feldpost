## Learnings

### Sticky headers need a bounded scroll parent

`position: sticky` on section intros only works when an ancestor owns `overflow-y: auto` **and** a definite height. The media section host is `flex-1 min-h-0`; the inner `.project-media-section` also needs `flex: 1 1 auto; min-height: 0; overflow-y: auto`. Without that, sticky silently fails (whole page scrolls instead).

### ‚ÄúLightbox like workspace pane‚Äù ‚âÝ grid ‚Üí detail route

Workspace **grid** click opens detail via `workspacePaneObserver.setDetailImageId` / `openDetailView`. **Lightbox** is owned by `media-detail-media-viewer` on preview click inside detail. For project sections (no workspace pane), replicate the viewer pattern: `pointerClick` ‚Üí `requestFullPreview` ‚Üí `PhotoLightboxComponent`. Use `isImageLikeStoragePath`, not `media_type` ‚Äî project media rows only carry `storagePath`.

### Dialog padding: `!p-2` fought the design system

`hlmDialogContent` CVA ships `p-6`. Forcing `!p-2` made the picker feel cramped vs upload panel. Upload panel body inset is actually smaller (`spacing-2`/`spacing-3`), but user expectation after migration was **more** air ‚Äî restoring `p-6` + upload-matched `h3`/subtitle typography was the right fix. Title typography must not duplicate `hlmDialogTitle` on `h3` (global heading scale lives in `styles.scss`).

### Archive UX: don‚Äôt treat ‚Äústatus changed‚Äù as ‚Äúleave route‚Äù

`onArchiveProject` had been clearing selection (`router.navigate(['/projects'])`) and `detailsPanelOpen`. Product intent: archive is a metadata change while reviewing the project. Fix: drop navigation/panel close; **switch `showArchived` to `true`** when archiving the open project so it appears in the archived sidebar list.

When the user toggles back to active projects, the **route and detail view stay on the archived project**, but the sidebar list is strictly status-filtered ‚Äî the archived row is **not** pinned into the active list. **Restore (vice versa):** `showArchived.set(false)` when restoring the open project (or any restore ‚Äî list always switches to active); route stays on the restored project; toggling back to archived hides it from the sidebar while detail remains.

**Correction (same day):** An earlier pin (`project.id === selectedId` in `sidebarProjects`) was wrong ‚Äî it showed archived projects in the active list. Status filter only; selection lives in the URL, not the sidebar filter.

### Toast stack interaction

Collapsed deck (max ~3) + hover expand was shipped in prior commit (`471976dd`). Dismiss button should stay on `hlmBtn ghost` so hover uses primary emphasis from `button-variants.ts` ‚Äî local SCSS hover on toast close was redundant and drifted from design system.

### Tests

`projects-page` archive test: `loadProjects` in constructor races with manual `projects.set()` ‚Äî use `mockResolvedValueOnce` and `await whenStable()` before asserting `currentProject()`. Avoid `detectChanges()` after toggling `detailsPanelOpen` without full project fixture unless `UploadManager`/panel inputs are wired.

## Mistakes / anti-patterns to avoid

- Applying `position: sticky` without verifying which element scrolls (use DevTools ‚Üí scroll container).
- Assuming workspace grid click opens lightbox (it opens detail; lightbox is one level deeper).
- Navigating away on archive/restore when only `status` and sidebar filter scope change.
- Overriding dialog `p-6` with `!p-2` without checking original picker baseline (`e670e0e4` had no override).
