# 2026-06-13 — Resolver Tray: Conceptual Reframe to Contradiction Resolution

## What changed

Introduced `docs/specs/service/media-upload-service/contradiction-resolution-model.md` — a new parent document that reframes the resolver tray as a **Constraint Satisfaction Problem (CSP)** with **Hierarchical Constraint Propagation** (Mackworth et al. 1985).

### Key insight

The resolver tray is not an "address picker" — it is a **contradiction resolver**. The system collects evidence from multiple sources (folder path, filename, EXIF, geocoder, gazetteer), and when those sources disagree or are insufficient, the tray asks the user to prune exactly one variable's domain. After each answer, the system propagates consequences before asking the next question.

### New taxonomy

Every `disambiguationKind` now maps to a contradiction class:

- **C1–C5**: Source contradictions (two sources disagree)
- **A1–A2**: Ambiguities (one source, multiple valid interpretations)
- **V1**: Validation failures (post-resolution inconsistency)

### Open gaps identified

- **G1**: Folder-to-folder sibling detection (e.g. `Wien/1200/` vs `St. Pölten/1200/`)
- **G2**: Decision scope by tier, not by groupingKey — admin-level resolution must fan out to all jobs sharing the constraint violation, not just the current group's jobIds
- **G3**: Post-resolution validation gate — Photon probe after admin conflict resolution before proceeding
- **G4**: Deferred resolution lifecycle — Skip must set explicit `deferred` status visible in Media Detail

## Mistakes / lessons

- The existing `applyAdminLevelConflictChoice` already does cascading (re-check after resolution, open new tray if still conflicted). The gap is **scope**: it operates on `group.jobIds`, but the decision at a city/postcode level should apply to all jobs with the same constraint violation across different groupingKeys (different streets under the same postcode).
- `project_address_a`/`project_address_b` are dead code (no producer) but types and stepper-FSM spec still reference them — marked as deprecated in the table.
