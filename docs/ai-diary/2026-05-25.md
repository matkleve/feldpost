## Invariants (lock in)

1. **Transition map is law:** if `transitionMediaDisplayState(current, next) === current`, the bug is the map or the call — not “Angular broken.”
2. **FSM table drives CSS:** each `data-state` sets opacity (or equivalent) for **every** layer; see `media-display.rendering-matrix.supplement.md#fsm-layer-opacity-matrix`.
3. **Separate enums:** `MediaItemState` (idle/selected/…) ≠ `MediaDisplayState` (idle/loading-surface-visible/…).
4. **Warm revisit legal path:** `loading-surface-visible` → … → `media-ready` → (`content-fade-in` → `content-visible` **or** `content-visible` when skip-fade) — all edges in `media-display-state.ts`.
5. **Cache work verification:** always **map → `/media` → leave → `/media` again** on grid intrinsic; optional row toggle compare.

## Upload N:N map regression (post folder-name fix)

**Symptom:** Detail shows address + green GPS chip; map has no marker; Show on map / tile map quiet action ineffective.

**Cause:** `resolve_media_location` always **appended** a new link; display hydration used **sort_order 0** (often address-only) while GPS lived on link 1. `viewport_markers` only sees rows with `latitude`, `longitude`, and `geog`. GPS chip used `media.latitude != null` only (not paired coords).

**Fix:** Migration `20260526140000_resolve_media_location_enrich_primary_link.sql` updates primary link via `update_location`; `find_or_create` merges coords on conflict. App: `displayLocationFromRows` prefers first zoomable row; chip uses `hasValidGpsCoordinates`; invalidate location cache after forward geocode; map refreshes viewport after `imageUploaded`.

**Apply DB:** `supabase db push` or run migration locally before expecting markers.
