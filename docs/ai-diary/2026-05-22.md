# AI diary — 2026-05-22

## Session focus

Media detail **Location** section: editable active GPS (map pick, paste coords, address search), in-place updates without reloading the workspace pane, resolving spinners, and a brief primary **text flash** on fields that actually changed. Parallel thread: user renaming **image → media** in code (agent paused when asked).

## What we shipped (end state)

- **Map pick from `/media`** — navigate to `/map`, place pin, return via `returnUrl`; `MediaDetailLocationSyncService` + deferred patch (`queueMicrotask`) so the detail view updates without `loadImage()` / `loading` skeleton.
- **GPS vs address** — active `latitude`/`longitude` in Location; EXIF stays readonly in Details; writes go through `resolve_media_location` RPC, not direct column updates when policy applies.
- **In-place refresh** — `refreshMediaLocationFields()` merges location columns only; `prepareLocationPatchAfterGpsChange()` clears stale street/city when coords change so old city names are not kept while skipping DB refresh.
- **Resolving UI** — `addressFieldsResolving` / `coordinatesResolving`; spinners on value cells; flash runs **after** spinners clear (not while spinner branch is shown).
- **Address search apply** — pick suggestion → RPC → patch all lines + coords → DB refresh → flash **changed** fields only (`collectLocationFieldChanges` with trimmed compare).
- **Flash** — `_detail-row-value-flash.scss`: primary **text** only (~2.75s, no row background); class on `.detail-row__field-trigger` / address trigger.

## What we learned (technical)

| Topic | Lesson |
| ----- | ------ |
| Full pane “reload” | `loadImage()` sets `loading=true` and hides the detail body. Location changes must use patch + `refreshMediaLocationFields`, never full load on map return. |
| Blank `/media` page | **Never write `media` (or other) signals inside an `effect` that reads them.** Schedule patch with `queueMicrotask` after the effect completes. |
| Stuck rows (`—`, `0,0°`) | If patch only has lat/lng + `address_label`, merge kept old street/city; `needsAddressResolutionAfterGps` was false when label existed. Fix: always sync from DB after GPS change; clear omitted address keys when coords change. |
| Flash invisible | Template shows spinner **instead of** value button while `addressFieldsResolving`; flash fired during spinner state. Fix: flash in `finally` after spinners off. |
| Flash on center | Row background + `color` on parent lost to child `.detail-row__field-trigger { color: var(--foreground) }`. Fix: animate color on the value element only. |
| City/country no flash | Same value after pick (e.g. still Vienna, Austria) → correct for **delta-only** flash; user confirmed not to flash unchanged fields. |
| `resolvePendingMediaItem` | Often `changed: false` when status already `resolved`; rely on `refreshMediaLocationFields` after RPC. |

## Session (afternoon): Media grid / `app-media-display` FSM

### Problem arc

- `/media` grid tiles were blank (content layer stuck at `opacity: 0` because `[data-state]` on `display: contents` host did not reach layer CSS).
- User wanted **shrink-then-reveal**: square placeholder → slot animates to photo ratio → then sharp image fades in (PDF previews too).
- Several wrong turns: invented `visibility` / `image--revealed` / `showSharpContent` (not in spec); duplicate `aspect-ratio` transition on viewport + slot; ratio emitted too early (instant jump); illegal FSM hop `ratio-known-contain → content-fade-in` (guard silently rejected).

### What we shipped (end state)

- **FSM strictly opacity-driven** per `docs/specs/component/media/media-display.md` choreography table — no extra visibility hacks.
- **Geometry ownership:** only `media-item__slot` owns `aspect-ratio` + transition + `border-radius` / clip; `app-media-display` + viewport fill the slot at `100%`; layers react to `data-state` on `.media-display__viewport`.
- **Ratio timing:** `--media-aspect-ratio` stays `1` during `loading-surface-visible`; `commitAspectRatioToSlot()` runs on `ratio-known-contain`; `transitionend` on **slot** (media-item forwards to `media-display.onSlotGeometryTransitionEnd()`).
- **Flow:** loading pulse (1:1) → slot shrink → `media-ready` → `content-fade-in` → `content-visible`.
- **Staged content** = optional blurred `warmPreviewUrl` bridge; grid often skips it visually (layer empty).
- **Token fix (what made shrink work):** `var(--transition-geometry)` is **not defined** (Phase 7 removed bridge aliases). Slot now uses `transition: aspect-ratio 300ms var(--motion-ease-out);` per `docs/design/tokens.md` §3.6 + `motion.md`.

### Key files touched

- `apps/web/src/app/shared/media-display/` (html, scss, ts, state transitions)
- `apps/web/src/app/shared/media-item/` (slot `transitionend`, preview fills slot, aspect ratio signal)
- `apps/web/src/app/shared/item-grid/item-grid.component.scss` (slot tokens for grid modes — earlier in thread)

### What we learned

| Topic | Lesson |
| ----- | ------ |
| **Simple first** | If a transition “does nothing”, check DevTools for **unset CSS variables** before refactoring FSM/TS. `--transition-geometry` was referenced in specs but never on `:root`. |
| FSM guards | `goTo()` that violates `MEDIA_DISPLAY_TRANSITIONS` **fails silently** — state stuck, CSS never applies (e.g. content layer stays `opacity: 0`). |
| Spec vs code | Choreography table allows only documented properties (`opacity`, slot `aspect-ratio`). No `visibility` on `<img>`. |
| Ownership | Parent slot cannot read child CSS; `aspectRatioChange` output is required to mirror ratio on `.media-item__slot`. |
| Staged vs content | **Loading** = grey pulse, no image. **Staged** = optional low-res blur. **Content** = final `resolvedUrl`. User-facing path is often loading → shrink → content only. |

### Agent mistake to avoid

- Jumped to complex FSM/CSS refactors while **`--transition-geometry` was empty** — one-line token fix was the actual shrink bug at the end.
- **Rule for future me:** grep `var(--…)` in `_typography-baseline.scss` / `tokens.md` **first**; specs may still name removed bridge tokens (`--transition-geometry`, `--transition-fade-in`).

### Note to future me (media grid)

- Read `docs/specs/component/media/media-display.md` + rendering-matrix supplement before layer/opacity edits.
- Slot transition: `300ms var(--motion-ease-out)` — not `--transition-geometry`.
- Never `ratio-known-contain → content-fade-in`; always `→ media-ready → content-fade-in`.
- Defer `aspectRatioChange.emit` until `triggerRatioTransition` / `commitAspectRatioToSlot`.
