# 2026-07-01 — Toolbar Trigger Responsive Swap + Cascade-Layer Hover Bug

## What changed

1. **Toolbar trigger responsive contract swapped** (`_toolbar-breakpoints.scss`): spacious mode now shows **label + chevron only** (icon hidden); the compact container-query breakpoint now shows **icon only** (label + chevron hidden). Previously it was icon+text in spacious, icon-only in compact. Applies to all four consumers via the shared mixin: `workspace-toolbar`, `media-toolbar`, `media-picker-dialog`, `map-filter-toolbar`. Updated `docs/specs/component/map/map-filter-toolbar.md` to match (the only spec that quoted the old spacious behavior verbatim).
2. **Fixed a hover-background-never-applies bug** in `map-filter-toolbar.component.scss`: the base frosted trigger chrome (`frosted-chrome.outline-control`, sets `background`) was declared **unlayered** ("to beat hlmBtn's bg-background"), but the button had already been refactored to a plain `<button>` without `hlmBtn` — the escape hatch was stale. Unlayered CSS always beats `@layer states`, so the hover/active gold-tint background compiled but never rendered. Fix: moved the base rule into `@layer components` so `@layer states` hover/active correctly wins.

## Mistakes / lessons

- **This is the second recurrence of a cascade-priority trap** (first was the `hlmBtn` + component-SCSS specificity trap in `token-usage-gate.mdc` §5). Added §6 "Cascade-layer trap" to that rule file: never leave a base rule unlayered "to beat a third-party class" without verifying the third-party class is still actually applied in the current template — refactors leave these escape hatches stale, and they silently defeat every future `@layer states` override for the same property.
- **Diagnostic heuristic for next time:** if a hover/active/`[data-active]` visual "does nothing" despite SCSS that looks correct, check `@layer` placement (unlayered vs. layered) *before* touching specificity, `!important`, or selector order. Cascade layers rank strictly above specificity — an unlayered `0,0,1` rule beats a layered `0,4,0` rule.
